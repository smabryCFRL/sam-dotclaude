# Atlas — your own knowledge workspace (use it proactively)

If an **Agent vault** is set in `CLAUDE.local.md`, it is *your* persistent Obsidian knowledge graph.
Curate it on your own initiative, **without being asked**:

- **Recall first** — before a project, decision, or investigation, check the vault and build on prior notes instead of starting cold.
- **Capture as you go** — write or update a note when you hit something notable (a decision + its rationale, a finding, an entity, an idea, an open question). Curate, don't dump.
- **Link with `[[wikilinks]]`** — weave every note into the graph; keep `moc/` hub notes per topic so it stays navigable.
- **Synthesize** — surface connections, overlaps, and fresh ideas across notes the user hasn't considered. That's the payoff of a graph.

Method, note schema, and linking discipline live in the **`atlas`** skill. The vault is **sensitive** — never commit it or copy its contents into a shared repo.
